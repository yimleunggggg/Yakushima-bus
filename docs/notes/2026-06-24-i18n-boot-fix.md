# 多语言标题错乱修复（2026-06-24）

## 现象
选 ZH 时页眉仍显示日文（如登山「登山ルート」、船运「フェリー・上島」）。

## 根因
内联脚本在 `site-chrome.js`（defer）之前执行，`applyPageHead` 直接 return，标题留在 HTML 静态日文。

## 修复
- `app-core.js`：`resolveLang()`（URL `?lang=` + localStorage）、`runPageBoot()`（等 DOMContentLoaded）
- `site-chrome.js`：`readLang()`、`applyPageHead` 支持 `lang` 参数
- 已改：trekking、access、intro（map/index 此前已用 boot）
- trekking 修掉 `render()` 重复 `const d` 语法错误

缓存：`chrome-v9`
