# 语言切换后导航回退中文

## 现象
切到 EN/JA 后点主导航，新页面又变中文（或 URL 仍带 `?lang=zh`）。

## 原因
1. `AppCore.setLang()` 会触发 `SiteChrome.refresh()`。
2. `refresh()` 里 `readLang()` 用的是 `resolveLang()`，**优先读 URL**，旧 `?lang=zh` 覆盖 localStorage。
3. 导航链接被重新渲染成 `?lang=zh`。

## 修复
- `site-chrome.js`：`readLang()` 改为 `AppCore.getLang()`（含内存 `_lang`）。
- `app-core.js`：`setLang()` 同步 `history.replaceState` 更新 URL 的 `lang` 参数。

## 缓存
- `app-core.js` → `core-v9`
- `site-chrome.js` → `chrome-v20`
- `site-chrome-nav-boot.js` → `chrome-v19`
