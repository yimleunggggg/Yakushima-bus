## 7. Core Web Vitals 技术分析

### 什么是 CLS（Cumulative Layout Shift，累计布局偏移）？

CLS 衡量页面加载过程中内容「意外移动」的程度。比如：图片加载后把下方文字往下推，用户准备点击的按钮突然跑位——这种体验就是 CLS。分数越低越好。

**评分标准：**
- 0 - 0.1：良好（绿色）
- 0.1 - 0.25：需要改进（橙色）
- > 0.25：差（红色）

**本站现状：**
- Desktop CLS **0.194**（橙色）：超标但不严重，主要影响 Google 的 CWV 评分
- Mobile CLS **0.495**（红色）：严重超标，几乎是阈值的 **2 倍**

Mobile CLS 0.495 意味着移动端用户经常会看到页面「抖动」——对于正在查看时刻表的用户体验极差。CLS 也是 Google 核心排名因子之一（Core Web Vitals），持续超标会对移动端搜索排名产生负面影响。

### LCP 3.9s（Mobile）问题链路

**什么是 LCP（Largest Contentful Paint，最大内容绘制）？** 指页面最大的可见元素（通常是主图片或大标题文字）完成渲染的时间。这是用户感知「页面加载好了」的关键节点。

**评分标准：**
- < 2.5s：良好
- 2.5 - 4.0s：需要改进
- > 4.0s：差

Mobile LCP 3.9s 处于「需要改进」上限边缘，几乎到差的临界点。问题链路：
1. **Render-blocking requests（-400ms mobile / -210ms desktop）**：CSS 或 JS 文件阻塞了渲染开始，浏览器必须等这些文件加载完才能开始绘制页面
2. **未优化的图片（-965 KiB）**：这是最大单项收益，超过 900KB 的图片未经优化，显著拖慢 LCP
3. Desktop LCP 仅 1.0s，说明移动网络条件（通常较慢）放大了这些问题

### 其他技术问题

| 问题 | 影响估算 | 说明 |
|---|---|---|
| Render-blocking requests | Mobile -400ms | CSS/JS 阻塞首屏渲染 |
| Reduce unused JavaScript | -64 KiB | 加载了未使用的 JS 代码 |
| Improve image delivery | -965 KiB | 最大单项，图片格式/压缩问题 |
| Use efficient cache lifetimes | -978 KiB | 静态资源缓存策略不足 |
| Minify CSS | -3 KiB | 小优化 |
| Minify JavaScript | -9 KiB | 中等优化 |
| Reduce unused CSS | -15 KiB | 中等优化 |
| Forced reflow（mobile only） | 影响 TBT | JS 强制同步布局计算，典型的 CLS 诱因 |
| Form elements 无 label | Accessibility | 影响无障碍访问和 Agentic Browsing 评分 |

## 8. Agentic Browsing 1/3 评分解读

**什么是 Agentic Browsing？** 这是 PageSpeed 的新维度，评估 AI Agent（如 GPT、Gemini、Claude）能否顺利「理解并操作」你的网页。AI Agent 靠 Accessibility Tree（无障碍树）来理解页面——类似于屏幕阅读器的逻辑。

**1/3 分意味着什么？**
- **Accessibility tree is not well-formed**：页面的可访问性树结构不规范，AI Agent 难以正确理解页面层级和可交互元素
- **CLS 问题**：页面在 AI Agent 操作过程中布局会偏移，导致 AI 点击错位
- 在 AI 搜索爬虫（如 GPTBot、ClaudeBot、Googlebot）抓取时，结构不规范的页面可能导致内容被错误理解或低优先级引用

本站已有 18 次 AI Assistant 渠道访问，说明部分 LLM 已推荐本站。将 Agentic Browsing 分数从 1/3 提升到 3/3 的主要手段：修复 CLS + 为所有表单元素添加正确的 `<label>` 标签。

## 9. 结构化数据 Warning 处理

**WARNING：WebApplication schema 中 `areaServed` 属性不被识别**

`areaServed` 是 `Organization` 或 `LocalBusiness` 类型支持的属性，但被误放在了 `WebApplication` schema 下。Google 的 Rich Results Test 认为这是无效属性。

**影响**：不会导致 Rich Results 失效，但属于「脏数据」，降低 Schema 精准度，也可能影响 AI 引用时对本站覆盖地区的理解。

**修复方案（两选一）：**
1. 移除 `WebApplication` 中的 `areaServed`，在同一页面添加独立的 `Organization` 或 `LocalBusiness` schema 来声明服务地区
2. 在 `WebApplication` 中改用 `applicationCategory` + `keywords` 来传达地理服务范围

## 10. 优先级排序优化建议清单

### P0（1-2 周内，移动端 CLS 修复）

**P0-1：修复 Mobile CLS 0.495（红色）**
- 原因：CLS 是 Core Web Vitals 排名因子，0.495 严重超标，直接影响移动端 Google 排名；移动端用户占比 46%
- 行动：开启 Chrome DevTools → Performance → Layout Shift，定位抖动元素（通常是未设定尺寸的图片、字体加载、动态注入的 DOM）；为图片标注 `width` 和 `height` 属性；避免 `font-display: auto`
- 预期收益：CLS 降到 0.1 以下后，Google 移动端排名可能提升 1-3 位；用户体验显著改善

**P0-2：优化图片交付（-965 KiB）**
- 原因：单项最大性能收益，直接压缩 Mobile LCP
- 行动：将 JPEG/PNG 转为 WebP/AVIF；添加 `loading="lazy"` + `decoding="async"`；设置 `srcset` 响应式图片
- 预期收益：Mobile LCP 从 3.9s 降至预计 2.5-3.0s

### P1（1 个月内，SEO 和可及性）

**P1-1：为所有表单元素添加 `<label>`**
- 原因：Accessibility 89 + Agentic Browsing 1/3 的主要扣分项；影响屏幕阅读器用户和 AI 爬虫
- 行动：检查时刻表选站下拉框、日期选择器等 `<select>` / `<input>` 是否有关联 `<label for="...">`
- 预期收益：Accessibility 可能升至 95+；Agentic Browsing 分数提升

**P1-2：消除 Render-blocking requests（-400ms mobile）**
- 行动：非关键 CSS 使用 `media="print" onload=`；非关键 JS 加 `defer` 或 `async`
- 预期收益：FCP 和 LCP 各降低约 0.3-0.4s

**P1-3：修复 Schema.org `areaServed` Warning**
- 行动：将 `areaServed` 移至独立的 `Organization` schema

**P1-4：加速新 URL 索引（`/fare/`、`/ferry/`、`/map/`）**
- 行动：在 GSC 中对新 URL 提交「请求编入索引」；确认 sitemap.xml 已更新并重新提交
- 原因：URL 迁移（2026-06-25 至 27）可能造成 2-4 周排名空缺期

**P1-5：设置 GA4 关键事件标记**
- 行动：在 GA4 管理中将 `timetable_search`、`fare_lookup`、`file_download` 标记为关键事件
- 原因：当前「关键事件: 0」导致无法在报告中看到转化指标；标记后可以量化功能实际价值

### P2（下一季度，增长机会）

**P2-1：JS/CSS 精简（-64 KiB JS / -15 KiB CSS）**
- 工具：使用 Bundle Analyzer 确认未使用代码来源；按需加载地图库

**P2-2：静态资源长期缓存（-978 KiB）**
- 行动：为 CSS/JS/图片设置 `Cache-Control: max-age=31536000, immutable`，文件名带哈希

**P2-3：中文流量开发**
- 行动：小红书/微信公众号发布一篇「屋久岛交通完整指南」，附上 yakushimabus.com 链接；这是成本最低的中文 SEO 突破口
- 预期：可将中国用户从 20.77% 进一步增长，且转化质量高（已有 43% 感兴趣率基础）

**P2-4：回访用户策略**
- 现状：回访用户仅 10%，对于旅行工具属正常，但屋久岛当地从业者（旅行社、民宿、租车店）是高频用户群体
- 行动：考虑在 about 页或页脚添加「适合旅行社/当地从业者的分享版」链接，引导 bookmark 行为
