## 事業者 UI（badge / 乘车券标签）

**唯一数据源**：`scripts/lib/operators.py` → 构建进 `data.js` 的 `operators` 字段。

| 字段 | 用途 |
|------|------|
| `short` | badge 简称 |
| `color` | badge 颜色 |
| `acceptsPass` / `acceptsIc` | 机器可读（筛选等扩展用） |
| `paymentTags` | 每条班次自动展示的标签文案 |

**前端**：`operator-ui.js` 读取 `BUS_DATA.operators`，`index.html` / `map.html` 共用，勿在 HTML 里硬编码公司名或标签。

**新增第三家公交时**：
1. 在 `operators.py` 加一条 + `parse_*.py` 路线设 `"operator": "新id"`
2. `python3 scripts/build_all.py --timetable`
3. UI 自动出 badge / 图例 / 标签，无需改 `operator-ui.js`

**乘车须知**长文仍在 `parse_pdf.py` 的 `info`；与 `paymentTags` 短标签互补，改支付规则时两处同步即可。

- 移动端 iframe PDF 兼容性（部分浏览器需「新窗口打开」）
